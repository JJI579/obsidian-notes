Stored Program Concept

John von Neumann invented the processor architecture which **stores a program in memory as instructions and executes them sequentially using the ALU, control unit and registers

CPU

Fetch-decode-execute cycle

Von Neumann Model

  • Input device loads programs and puts data into memory (Stored Program Concept)
  • Processes with CPU / Memory
  • Result to output device

Common types of CPU instruction set

CISC - Complex Instruction Set - Such as Intel x86 One instruction can require the CPU to do more than one clock cycles for it to compute and execute it RISC - Reduced Instruction Set - Such as ARM Low Level instructions, and per instruction only requires one clock cycle internal activity of CPU synchronised by fast clock aka hertz

System Bus

Bus is collection of wires - provides communication between various components of the motherboard - With no bus - have to directly connect all components in mesh-style factor - Sender places data on the bus and receiver takes it off

Lines of bus

  • Address lines - specifies memory address or device to be accessed
  • Data lines - Carry the actual data to be transferred
  • Control lines - tells receiver what to do with the data
