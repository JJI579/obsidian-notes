Adversial Search (Game Playing)

Search that can be applied to playing games - perfect play - minimax algorithm - \(\alpha \beta\) pruning How to operate with limited resources / heuristics

Types of Games - Fully observable environments (Games played in this environment is calld perfect information) - Partially observable information (Games in this environment is called imperfect information)

Requirements for considering an AI-based agent for a game - Perfect information - zero sum (whatever +1 for you it is -1 to the opponent) - Deterministic

Differences between normal search techniques

  • Set of goal states replaced by a utility function

Introducing MIN and MAX

MIN & MAX are two players

  • MAX wants to win (maximises the utility)
  • MIN wants MAX to lose (minimise utility for MAX)
  • MIN is the opponent in this circumstance

Both players will play to the best of their ability - Have to consider minmax value of each state: which is the utility of a state given that both players play optimally (to the best of their ability)

Terms

Terminal State - Value provided by utility MAX-State - maximum of utilities of its sucessor states MIN-State - minimum of utilities of its successor states

Consider the search like a tree, the states of successors can be determined from from descendants of the terminal states (finalising games) ![[Pasted image 20241011144004.png]]

Properties

Time Complexity - can be implemented as DFS so that time complexity is \(b^m\) [[Key Terms#|b]] [[Key Terms#|m]]

Space Complexity - can be implemented as DFS so that space complexity is \(b*m\)

Problems

due to the space and time complexity with the game’s functionality, an example being chess where the calculations for one move is \(10^{134}\)

Which is why we implement this… ## \(\alpha \beta\) Pruning this is based on redacting redundant information / performance requirements through not performing unnecessary calculations when during the calculation it has been identified that it will succeed or fail

Cutoff & Heuristics

Cutoff - suggests to implement a depth limit during the search in and once complete utilise heuristic evaluation function to identify how effective the next potential state is.

Heuristics are used rather than a utility function

Cutoff is defined recursively to iterate through the successor states and identifies the best potential path with the limited knowledge it has

If cutoff is implemented the strategy changes from DFS to DLS
